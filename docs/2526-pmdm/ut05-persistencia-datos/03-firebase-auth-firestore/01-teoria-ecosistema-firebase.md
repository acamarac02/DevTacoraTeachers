Alcanzamos el almacenamiento en la nube en esta sección introduciendo **Firebase**, una plataforma Backend-as-a-Service (BaaS) de Google. En este apartado explicaremos qué es, para qué sirve y desgranaremos los dos grandes pilares a tratar:
1. **Firebase Authentication**: Solución para registrar y mantener sesiones de usuario.
2. **Firebase Firestore**: Base de datos NoSQL, entendiendo el concepto de Colecciones y Documentos frente a las relaciones tradicionales de SQLite vistas de Room. Se explicarán las bases del modelado de datos no relacional.
