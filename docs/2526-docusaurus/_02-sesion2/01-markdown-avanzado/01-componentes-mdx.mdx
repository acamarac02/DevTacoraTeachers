El formato **MDX** es una de las características más potentes que nos brinda Docusaurus. MDX nos permite insertar componentes de React directamente dentro del documento Markdown estándar. Esta capacidad es clave para poder añadir estructuras y utilidades enriquecidas y avanzar nuestro material hacia una plataforma interactiva.

## Implementación de Pestañas (Tabs)

Para agrupar códigos fuente temporalmente sin saturar una lección, Docusaurus incluye nativamente los componentes `<Tabs>` y `<TabItem>`. 

Antes de utilizarlos en tu lección, es obligatorio importarlos al principio del documento (justo debajo del *frontmatter* en tu archivo `.mdx` o `.md`):

```mdx 
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
```

## Casos de uso docente

A continuación vemos un par de implementaciones típicas para vuestras clases:

### Pseudocódigo vs Código

En asignaturas de introducción a programación, resulta extremadamente productivo que el alumno pueda bascular entre el diseño algorítmico y la implementación en el lenguaje meta:

````mdx
<Tabs>
  <TabItem value="pseudo" label="Pseudocódigo" default>
    ```text
    INICIO
      Declarar variable i 
      Bucle i de 1 a 10
        Imprimir i
      Fin Bucle
    FIN
    ```
  </TabItem>
  <TabItem value="java" label="Java">
    ```java
    for(int i = 1; i <= 10; i++) {
        System.out.println(i);
    }
    ```
  </TabItem>
</Tabs>
````

**Resultado en el navegador del código anterior:**

<Tabs>
  <TabItem value="pseudo" label="Pseudocódigo" default>

```text
INICIO
  Declarar variable i 
  Bucle i de 1 a 10
    Imprimir i
  Fin Bucle
FIN
```

  </TabItem>
  <TabItem value="java" label="Java">

```java
for(int i = 1; i <= 10; i++) {
    System.out.println(i);
}
```

  </TabItem>
</Tabs>

### Tecnologías Equivalentes

De igual manera, en formación más avanzada como DAM donde saltamos entre lenguajes (por ej. desarrollo Android), las pestañas eliminan la necesidad de duplicar manuales:

````mdx
<Tabs>
  <TabItem value="java" label="Java" default>
    ```java
    Button button = findViewById(R.id.myButton);
    button.setOnClickListener(new View.OnClickListener() {
        @Override
        public void onClick(View v) {
            // Acción en Java
        }
    });
    ```
  </TabItem>
  <TabItem value="kotlin" label="Kotlin">
    ```kotlin
    val button: Button = findViewById(R.id.myButton)
    button.setOnClickListener {
        // Acción en Kotlin
    }
    ```
  </TabItem>
</Tabs>
````

**Resultado en el navegador:**

<Tabs>
  <TabItem value="java" label="Java" default>

```java
Button button = findViewById(R.id.myButton);
button.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View v) {
        // Acción en Java
    }
});
```

  </TabItem>
  <TabItem value="kotlin" label="Kotlin">

```kotlin
val button: Button = findViewById(R.id.myButton)
button.setOnClickListener {
    // Acción en Kotlin
}
```

  </TabItem>
</Tabs>

:::tip[Comportamiento en navegador]
Docusaurus recordará en el navegador del alumno cuál fue la pestaña que seleccionó la última vez, aplicando esa selección en futuras vistas si usan el mismo identificador (`value`).
:::
