# ¿Por qué Docs-as-Code?

Como docentes, solemos dedicar hasta un **40% de nuestro tiempo** a tareas de formato: pelear con Powerpoint, Word o PDFs para que los apuntes se vean "bien".

El modelo **Docs-as-Code** (Documentación como Código) propone una filosofía diferente.

## El Concepto Central

Escribir documentación utilizando las mismas herramientas que usamos para programar:
*   **Texto plano (Markdown)**: Para el contenido.
*   **Control de versiones (Git)**: Para el histórico de cambios.
*   **Automatización (Docusaurus)**: Para el diseño y la publicación.

## Ventajas para el Docente

| Ventaja | Descripción |
| :--- | :--- |
| **Enfoque Total** | Te centras en el texto. Docusaurus se encarga de que sea bonito y accesible. |
| **Código Interactivo** | Facilita compartir código fuente con los alumnos, permitiéndoles copiarlo y probarlo al instante con un clic. |
| **Portabilidad** | Los archivos son texto plano. Nunca perderás el contenido por cambios de formato de software propietario. |
| **Multidispositivo** | Tus alumnos tendrán una web que carga perfecto en móviles, tablets y ordenadores. |
| **Buscador Integrado** | Permite al alumnado encontrar cualquier concepto en segundos. |

## Integración con IA

En este curso utilizaremos **Antigravity**. Gracias a que trabajamos en texto plano, los agentes de IA pueden entendernos mucho mejor y ayudarnos a:
*   Generar casos prácticos a partir de una explicación.
*   Simplificar conceptos complejos.
*   Corregir erratas en bloque.
